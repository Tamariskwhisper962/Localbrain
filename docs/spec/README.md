# localbrain — 범용 로컬 RAG MCP 도구 (스펙)

> 작성일: 2026-05-30 · 성격: 설계/산정 스펙 (구현 전 단계)

특정 폴더·파일을 경로로 추가해 인덱싱하고, **MCP로 검색**을 제공하며,
**사용자 질의 자체를 임베딩·인덱싱**하고 **클러스터링**해 FAQ·지식공백을 발견하는
**범용 로컬 RAG 도구**의 설계와 작업량 산정.

## 무엇을 만드는가 — 4개 축

| # | 기능 | 개념(localbrain index.html 참조) |
|---|------|------------------------------|
| **A** | 경로(폴더/파일) 추가 → 청킹·임베딩·인덱싱 | 기본 RAG 인덱싱 (§3) |
| **B** | MCP로 검색/관리 도구 노출 (Claude가 호출) | RAG를 MCP로 감싸기 (§4·§6) |
| **C** | 사용자 질의 자체도 임베딩해 저장·인덱싱 | 자가개선 루프 ① (§5) |
| **D** | 질의 임베딩 클러스터링 → FAQ·인기주제·지식공백 | 자가개선 루프 ② (§5) |

**핵심 통찰**: A·B는 commodity(이미 OSS 다수 — mcp-local-rag·LEANN 등). 차별화·노동은 **C·D 레이어**.

## 작업량 요약 (1인, Python, 로컬 임베딩, 개인 규모)

| 시나리오 | 범위 | 작업량 |
|---------|------|--------|
| MVP (헤드리스, 4축 거친 버전) | md/txt 위주, 증분 기본, 군집 통계 단순, UI 없음 | **약 1주 (5~7일)** |
| **MVP + 간단 운영 UI** ⭐ | 위 + 소스 관리·수동 인덱싱·진행상황·검색 테스트·모델 교체 웹 4탭 | **약 1.5~2주** |
| 견고·범용 (실사용 품질) | 멀티포맷·하이브리드·증분 견고·인사이트 탭·패키징 | **약 2.5~3.5주** |
| OSS 기반 (A·B 재활용) + C·D + UI 신규 | 기존 RAG MCP 채택 후 질의레이어·UI만 신규 | **약 1.5주, 리스크 최저** |

### 확정 설계 요구사항
- **모델 교체 용이성(1급)**: 임베딩을 Provider 인터페이스 뒤로 숨기고 config로 교체, 모델별 컬렉션 버전 분리 + 원자적 전환. 재인덱싱 비용은 감수 (청크 원문 보관으로 재파싱 없이 재임베딩). → [`architecture.md`](architecture.md) §9, [`embedding-runtime.md`](embedding-runtime.md)
- **간단 운영 UI(초기 범위)**: 소스 추가 · 수동 인덱싱 실행 · 진행상황 · 검색 테스트 · 모델 교체. → [`ui-review.md`](ui-review.md)

작업량을 흔드는 변수: 파일 포맷 범위(최대 scope creep), 증분 정확성, 클러스터링 실효성, 한국어 임베딩, 규모, 패키징. 자세히는 [`architecture.md`](architecture.md) §리스크.
